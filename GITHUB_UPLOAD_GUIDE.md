# 📤 GitHub Upload Guide — ChatWave

This document explains **which files should be uploaded to GitHub** and **which files must be ignored** to keep the repository clean and secure.

---

## 📁 Files to Upload

### Root Directory

```
Chat App/
├── backend/
├── frontend/
├── README.md
├── IMPLEMENTATION.md
├── GITHUB_UPLOAD_GUIDE.md
└── .gitignore
```

### Backend Files

```
backend/
├── server.js
├── package.json
├── .gitignore
├── .env.example        ← Template only, NOT .env
├── config/db.js
├── models/User.js
├── models/Message.js
├── middleware/auth.js
├── controllers/authController.js
├── routes/auth.js
└── routes/messages.js
```

### Frontend Files

```
frontend/
├── src/
├── angular.json
├── tsconfig.json
├── package.json
├── package-lock.json
└── .gitignore
```

---

## 🚫 Files to NEVER Upload

| File            | Reason                                  |
| --------------- | --------------------------------------- |
| `.env`          | Contains DB password & JWT secret       |
| `node_modules/` | Huge; install with `npm install`        |
| `dist/`         | Build output, regenerated by `ng build` |
| `*.log`         | Temporary log files                     |
| `.DS_Store`     | macOS metadata                          |

---

## 🔒 Environment Variables

Never upload `.env`. Upload `.env.example` instead:

```env
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_secret_key
```

## 🚀 Upload Commands

```bash
# Inside the Chat App folder:
git init
git add .
git commit -m "Initial commit - ChatWave Real-Time Chat App"
git remote add origin https://github.com/naivedhP2518/Chat-Web.git
git branch -M main
git push -u origin main
```
